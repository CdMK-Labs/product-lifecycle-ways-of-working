## What this page is for

Six deliverable packs cover the full product lifecycle. Each pack maps to a lifecycle stage and a financial stage.

Open a pack card to access the guidance page and download the starter files.

## Deliverable packs

- **01 [opportunity-brief] Opportunity Brief** — Frames the problem, the expected value and why it matters now. Used to decide whether to move into discovery.
- **02 [discovery-pack] Discovery Pack** — Captures what was learned in discovery: problem understanding, options explored, risks and a recommendation on next steps.
- **03 [business-case-and-delivery-decision-pack] Business Case & Delivery Decision Pack** — The main approval deliverable. Brings together the value case, cost case and delivery plan for commitment and sign off.
- **04 [delivery-pack] Delivery Pack** — Tracks scope, progress, quality and dependencies throughout build and delivery. Keep it updated as work progresses.
- **05 [release-and-readiness-pack] Release & Readiness Pack** — Confirms the change is ready for go live: release readiness, operational setup and adoption preparation.
- **06 [product-review-pack] Product Review Pack** — The recurring live product review covering service health, usage and adoption signals, and improvement decisions.

Starter packs are available to download from each pack page.

## How to use this model

Start with the stage you are in. Open the relevant pack and use it as the baseline for that stage.

- Do not create more material than the stage requires
- Update existing deliverables rather than creating new ones
- Link to source tools, dashboards or working notes instead of duplicating content
- Use the stage exit criteria to define what is good enough

## Principles

- Document to support decisions, delivery and operations, not for its own sake
- Keep deliverables as light as possible for the stage
- Treat deliverables as information objects, not fixed file types
- Use one source of truth where possible
- Keep the lifecycle model and financial model aligned
