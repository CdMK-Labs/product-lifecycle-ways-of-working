Release & Readiness Pack

What this pack is for

Used in the Deploy stage to confirm the change is ready for go-live and coordinate the release. Covers deployment, rollout, support setup, monitoring, communication and the go-live decision.

Not only a technical checklist. The outcome is either go-live, or a decision to stabilise, rework or rollback.

What's in this pack

  • 01-Release-and-Readiness-Pack.docx — main pack. Covers the release summary, deployment plan, rollout plan, readiness checklist, support setup, monitoring, communication and training, rollback plan and go-live decision.

When to use this pack

Use in Deploy, when changes are ready to move toward production and deployment, support and user readiness need to be confirmed.

  • quality, security and non-functional checks have reached the level needed for go-live
  • support, monitoring and operational readiness need to be confirmed
  • rollout, communication and training activities need to be coordinated
  • the team needs a clear basis for the go-live decision
